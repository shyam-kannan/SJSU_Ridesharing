# Booking Service

## Purpose

The Booking Service handles all booking and reservation operations:

- Booking creation and management
- Booking requests from passengers
- Booking approval/rejection by drivers
- Seat reservation and allocation
- Booking status tracking (pending, confirmed, cancelled)
- Booking modifications
- Cancellation handling and policies
- Waitlist management
- Booking history

## Technology

- Node.js/TypeScript or Python
- PostgreSQL for booking data
- Redis for real-time booking updates
- Event-driven architecture for notifications

## Key Features

- Request and confirm bookings
- Real-time seat availability
- Automatic and manual booking approval
- Cancellation policies and penalties
- Waitlist functionality
- Booking conflict resolution
- Integration with payment service
